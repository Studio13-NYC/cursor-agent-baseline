# Test Plan

## Purpose
Define the most useful tests for the changed behavior.

## Inputs
- feature or fix
- impacted layers
- user-visible risk
- available harnesses

## Process
1. identify critical user paths
2. include edge and failure cases
3. choose the lowest useful test level
4. add cross-layer checks if contracts changed

## Outputs
- prioritized test list
- scope by test level
- known gaps
