# Endpoint Scaffold

## Purpose
Create a clear starting point for a typed API endpoint.

## Inputs
- route purpose
- request/response shape
- auth and validation requirements
- backing service expectations

## Process
1. define route and method
2. define request and response types
3. add validation boundary
4. keep controller thin
5. connect to service layer
6. shape consistent errors

## Outputs
- endpoint skeleton
- type definitions
- validation plan
