Design transport shapes that are explicit, stable, and easy to consume.

## Inputs
- use case
- UI consumer needs
- service output shape

## Process
1. define request fields
2. define response fields
3. exclude persistence internals
4. ensure names reflect domain meaning
5. keep optionality intentional

## Outputs

- notes on additive vs breaking changes
