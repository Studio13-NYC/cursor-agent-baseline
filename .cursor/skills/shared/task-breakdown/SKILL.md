# Task Breakdown

## Purpose
Break a request into the smallest useful work units without losing intent.

## Inputs
- user goal
- constraints
- affected layers
- known dependencies

## Process
1. identify the primary outcome
2. separate required work from optional improvements
3. group tasks by owning agent or layer
4. order tasks by dependency
5. surface open questions only if they block progress

## Outputs
- ordered work plan
- ownership by agent
- dependency notes
