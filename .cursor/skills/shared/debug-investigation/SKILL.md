# Debug Investigation

## Purpose
Find the most likely cause quickly and reduce wasted changes.

## Inputs
- failing behavior
- reproduction steps
- logs, errors, or screenshots
- recent changes

## Process
1. confirm the exact symptom
2. isolate the failing layer
3. test the highest-probability causes first
4. inspect recent changes and boundary assumptions
5. propose the smallest corrective change
6. define verification steps

## Outputs
- likely root cause
- smallest safe fix
- verification checklist
