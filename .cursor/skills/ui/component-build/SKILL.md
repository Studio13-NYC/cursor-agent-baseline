# Component Build

## Purpose
Create or update a UI component with clear structure and predictable behavior.

## Inputs
- component purpose
- required props and data
- visual constraints
- interaction states

## Process
1. define the component contract
2. structure semantic markup
3. implement state and events
4. apply maintainable styling
5. handle loading, empty, and error states
6. verify accessibility basics

## Outputs
- component implementation
- prop/state notes
- behavior checklist
